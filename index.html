Description

The game bundle is a device that contains two pre-loaded games and a menu interface. The games contained in the bundle are three difficulties of mazes navigated using the tilt orientation of the CC3200 and hangman controlled using an IR remote. The IR remote is also used to interact with the menu screens.

Design

System Architecture

As shown in the system flowchart, we use one of the CC3200 boards for the closed feedback loop for maintaining concentration in the environment. The board will read values with I2C protocol through an ADC which reads values from the thermistor and the TDS meter sensor. The board will also periodically read user-defined thresholds from the AWS cloud using RESTful APIs. When the sensory values read outside of the user-defined thresholds, the board will activate the motor control function to pump either water or nutrient solution to bring the concentration back within the thresholds. Meanwhile, another CC3200 board is frequently reading from the AWS cloud to present the current TDS reading and user-defined threshold to the user on an OLED through SPI protocols. To adjust the thresholds, the user can either do it remotely or locally by using a TV remote to type the number into the IR receiver. The adjustments will be updated to the AWS and if the user updates remotely, the local CC3200 board will update the values in the next synchronization.

Functional Specification

Our system works based on the following state diagram. The device will periodically monitor the temperature and the electrical conductivity (EC) of the solution and convert the values into a TDS value using a calibration curve. At the same time, the device will check for threshold inputs, both over the AWS shadow and via manual input on the IR receiver. It will compare the TDS with the lower and upper thresholds set by the user. If the value is within the thresholds, it will stay in the normal state, update the OLED, and wait for the next monitoring period. If the TDS value is too low or too high, the device will move into the corresponding state to add water or nutrient solution to the system to bring the values within range. This will be done in bursts as the device will move back to the normal state to wait for the system to re-equilibrate before repeating the process.

Implementation

CC3200-LAUNCHXL Evaluation Board

CC3200-LAUNCHXL Evaluation Board

Our implementation heavily utilizes the CC3200-LAUNCHXL boards. This board was selected due to its integrated Wi-Fi module which is essential for communication with the AWS IoT Core. The board contains an ARM Cortex-M4 core and supports multiple communication protocols, including UART, SPI, and I2C. The Wi-Fi module on the CC3200 is used to communicate with the AWS IoT Core and allows for remote monitoring and control of the hydroponic system. The microcontroller handles sensor data acquisition, motor control, and user interface operations.

AWS IoT Core

AWS IoT Core

The AWS IoT Core acts as a bridge between the hardware and the user interface. It stores the thresholds and sensor data and facilitates communication between the user's remote commands and the system. The CC3200-LAUNCHXL master board periodically publishes the sensor data to the AWS IoT Core and subscribes to the shadow document to update its internal threshold values. This ensures that any remote changes made by the user are reflected in the deviceâ€™s operation.

OLED Display

OLED Display

The OLED display is used to present the current TDS reading and user-defined thresholds to the user. It is connected to the CC3200-LAUNCHXL board via SPI protocol. The display provides a clear and concise interface for the user to monitor the system's status and make necessary adjustments.

IR Receiver

IR Receiver

The IR receiver allows the user to interact with the system locally. By using a TV remote, the user can input new threshold values or command the system to perform certain actions. The IR receiver decodes the signals from the remote and sends the corresponding commands to the CC3200-LAUNCHXL board for processing.

Challenges

During the implementation of our hydroponic system, we faced several challenges. One major challenge was ensuring accurate and reliable communication between the sensors, microcontroller, and AWS IoT Core. This required careful calibration and testing to ensure the system operated correctly under various conditions. Another challenge was optimizing the power consumption of the system to ensure it could operate efficiently over extended periods without frequent maintenance.

Future Work

For future work, we plan to further refine the system's user interface and add additional features such as automated pH control and integration with other smart home devices. We also aim to improve the system's scalability to support larger hydroponic setups and enhance the accuracy of sensor data through advanced calibration techniques.
